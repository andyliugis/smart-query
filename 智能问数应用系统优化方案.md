### 1.2 引入数据可视化（ECharts）

```typescript
// chart-recommender.ts
export class ChartRecommender {
  static recommendChartType(columns: string[], data: any[]): ChartType {
    const numericColumns = this.findNumericColumns(columns, data);
    const categoricalColumns = this.findCategoricalColumns(columns, data);
    const timeColumns = this.findTimeColumns(columns, data);
    
    if (timeColumns.length >= 1 && numericColumns.length >= 1) {
      return 'line'; // 折线图 - 时间趋势
    } else if (categoricalColumns.length >= 1 && numericColumns.length >= 1) {
      if (categoricalColumns.length === 1) {
        return 'bar'; // 柱状图 - 对比
      } else {
        return 'groupedBar'; // 分组柱状图
      }
    } else if (categoricalColumns.length === 1 && numericColumns.length >= 1) {
      return 'pie'; // 饼图 - 占比
    }
    
    return 'table'; // 默认表格
  }
}
```

### 1.3 流式响应重构（SSE）

**借鉴 Vanna 的组件流式设计**：

**后端实现**：
```java
// ChatStreamController.java
@RestController
@RequestMapping("/api/chat")
public class ChatStreamController {
    
    @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<ChatComponent>> chatStream(
            @RequestParam String question,
            @RequestParam(required = false) Long sessionId,
            Principal principal) {
        
        return Flux.create(sink -> {
            try {
                // 1. 状态更新: 分析中...
                sink.next(statusEvent("正在分析您的问题..."));
                
                // 2. 生成 SQL
                String sql = llmService.generateSQL(question, sessionId);
                sink.next(sqlEvent(sql));
                
                // 3. 状态更新: 执行查询...
                sink.next(statusEvent("正在执行查询..."));
                
                // 4. 执行 SQL
                Map<String, Object> result = sqlExecutorService.executeQuery(sql);
                
                // 5. 发送表格数据
                sink.next(tableEvent(result));
                
                // 6. 状态更新: 生成图表...
                sink.next(statusEvent("正在生成可视化图表..."));
                
                // 7. 生成图表数据
                ChartData chart = chartService.generate(result);
                sink.next(chartEvent(chart));
                
                // 8. 生成总结
                String summary = llmService.generateSummary(result);
                sink.next(textEvent(summary));
                
                sink.complete();
                
            } catch (Exception e) {
                sink.error(e);
            }
        });
    }
}
```

**前端实现**：
```typescript
// chat.service.ts
export function streamChat(question: string, sessionId?: number) {
  const eventSource = new EventSource(
    `/api/chat/stream?question=${encodeURIComponent(question)}` + 
    (sessionId ? `&sessionId=${sessionId}` : '')
  );
  
  return new Observable<ChatComponent>(observer => {
    eventSource.onmessage = (event) => {
      const component = JSON.parse(event.data);
      observer.next(component);
    };
    eventSource.onerror = (error) => {
      observer.error(error);
      eventSource.close();
    };
    return () => eventSource.close();
  });
}
```

### 2.2 用户感知架构（借鉴 Vanna）

**核心概念**：用户身份贯穿整个请求链路

```java
// UserContext.java
@Data
public class UserContext {
    private String userId;
    private String email;
    private List<String> groupMemberships;
    private Map<String, Object> metadata;
    
    public static UserContext fromAuthentication(Authentication auth) {
        // 从 Spring Security 提取
    }
}

// 工具抽象（Vanna 风格）
public interface Tool<T> {
    String getName();
    String getDescription();
    List<String> getAccessGroups();
    ToolResult execute(UserContext user, T args);
}

// 工具注册器
@Component
public class ToolRegistry {
    private Map<String, Tool<?>> tools = new ConcurrentHashMap<>();
    
    public ToolResult execute(String toolName, UserContext user, Map<String, Object> args) {
        Tool<?> tool = tools.get(toolName);
        
        // 权限检查
        if (!hasAccess(user, tool)) {
            throw new AccessDeniedException();
        }
        
        return tool.execute(user, args);
    }
}
```

## 🎯 总结

我们的当前基础很好（Spring Boot + Vue 3 + Element Plus），只需要：

1. **第一阶段**：专注于用户体验（布局、可视化、流式）
2. **第二阶段**：增强功能（多模型、数据源、术语库）
3. **第三阶段**：企业级特性（工作空间、看板、深度分析）

**核心原则**：
- 优先做用户能直接感知到的体验优化
- 逐步引入优秀架构，不要大改
- 保持代码清晰，易于维护和扩展

这就是我们的优化路线图！
