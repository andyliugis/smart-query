# 智能问数 - 开发思路与项目规划

## 项目定位

开发一款"智能问数"应用：用户用自然语言提问，系统通过 LLM 将问题转为 SQL 查询，执行后返回结构化结果。核心能力是 Text2SQL + 语义层 + 智能问答。

## 技术栈

| 层面 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | Vue 3 + Vite + TypeScript + Element Plus + ECharts | SPA 对话式界面 |
| 后端 | Spring Boot 3.4 + JDK 17 + Maven + Spring AI 1.0.0 | 核心引擎 |
| LLM | 智谱 GLM-4-Flash（Spring AI 原生支持） | 通过 `spring-ai-starter-model-zhipuai` 接入 |
| 数据库 | H2 内嵌（MVP 演示） | 后续扩展至 MySQL/ClickHouse |

## 开发路线（四步走）

### 第一步: MVP 骨架 + 基础 Text2SQL [已完成]

搭建前后端项目，跑通最基础的"自然语言 -> SQL -> 查询结果"流程。

**后端核心**:
- `QueryController` - POST /api/query 接口
- `LLMService` - 调用智谱 GLM 将自然语言转 SQL（ChatClient + System Prompt）
- `SQLExecutorService` - 安全执行 SQL，只允许 SELECT，拦截危险操作
- `WebConfig` - 跨域配置

**前端核心**:
- `ChatView.vue` - 类 ChatGPT 对话界面，消息气泡 + SQL 代码块 + 结果表格
- `queryApi.ts` - axios 封装后端接口
- `vite.config.ts` - /api 代理到后端 8080

**演示数据**:
- `sales_data` 销售数据表（25 条记录，含产品/类别/金额/数量/日期/区域/销售人员）

### 第二步: 语义层 - 自定义表模型、指标、维度 [已完成]

在物理数据库和 LLM 之间建立语义层，将"表/字段"映射为业务语言"指标/维度"。

**核心思路**:
```
用户提问 -> 语义层(指标+维度+表结构) -> 动态组装 Prompt -> LLM 生成 SQL -> 执行
```

**数据模型（5 张元数据表）**:
- `table_metadata` - 表元数据，支持自动探测 + 人工标注
- `column_metadata` - 字段元数据，含角色标注（dimension/metric/none）
- `metric_definition` - 指标定义（名称/表达式/聚合方式/单位）
- `dimension_definition` - 维度定义（名称/关联字段/枚举值）
- `datasource_config` - 数据源配置（为多数据源预留）

**核心服务**:
- `MetadataService` - JDBC DatabaseMetaData 自动同步表结构，管理语义层 CRUD
- `PromptBuilder` - 从语义层动态组装 System Prompt（表结构 + 指标列表 + 维度列表 + 规则）
- `MetadataController` - 元数据管理 REST API

**管理 API**:
- `POST /api/metadata/sync` - 从数据库自动同步表结构
- `GET /api/metadata/tables` - 查询表和字段
- `PUT /api/metadata/columns/{id}` - 更新字段标注
- `GET/POST/DELETE /api/metadata/metrics` - 指标管理
- `GET/POST/DELETE /api/metadata/dimensions` - 维度管理

**预置语义层数据**:
- 指标: 销售额(SUM)、销售数量(SUM)、平均单价(AVG)、最高单价(MAX)、销售笔数(COUNT)
- 维度: 产品名称、产品类别、销售区域、销售人员、销售日期

### 第三步: 前端管理界面 + RAG 增强 [待开发]

**3.1 元数据管理前端**:
- 表/字段浏览页面，可视化展示表结构
- 字段角色标注交互（拖拽或下拉选择 dimension/metric/none）
- 指标/维度的增删改查界面
- 数据源配置页面（连接信息管理、连通性测试）

**3.2 RAG 增强 SQL 生成**:
- 引入向量数据库（如 Milvus 或 pgvector）
- 存储历史优质 SQL 样本 + Few-shot 示例
- 用户提问时先检索相似样本，注入 Prompt 提升准确率
- 支持 SQL 模板库（常见查询模式的参数化模板）

**3.3 查询增强**:
- 查询历史记录与收藏
- 结果可视化（ECharts 图表：柱状图/饼图/折线图）
- SQL 执行计划展示

### 第四步: 生产化 + 体验优化 [待开发]

**4.1 多数据源支持**:
- 对接 MySQL / PostgreSQL / ClickHouse / Hive 等真实数据源
- 数据源连接池管理
- 跨数据源查询

**4.2 权限与安全**:
- 用户认证（JWT / OAuth2）
- 数据权限控制（行级/列级权限）
- 查询审计日志

**4.3 体验优化**:
- SSE 流式输出（SQL 生成过程实时展示）
- 多轮对话（上下文记忆，支持追问和细化）
- 查询纠错（LLM 自检生成的 SQL，执行失败时自动修复重试）
- 智能推荐（根据表结构推荐常见问题）

## 当前项目结构

```
myqoder/
├── 开发思路                          # 开发笔记
├── smart-query-backend/              # 后端 (Spring Boot 3)
│   ├── pom.xml
│   └── src/main/java/com/smartquery/
│       ├── SmartQueryApplication.java    # 启动类
│       ├── config/
│       │   └── WebConfig.java            # 跨域配置
│       ├── controller/
│       │   ├── QueryController.java      # POST /api/query 查询接口
│       │   └── MetadataController.java   # 元数据管理 API
│       ├── model/
│       │   ├── QueryRequest.java         # 查询请求 DTO
│       │   ├── QueryResponse.java        # 查询响应 DTO
│       │   ├── DatasourceConfig.java     # 数据源配置实体
│       │   ├── TableMetadata.java        # 表元数据实体
│       │   ├── ColumnMetadata.java       # 字段元数据实体
│       │   ├── MetricDefinition.java     # 指标定义实体
│       │   └── DimensionDefinition.java  # 维度定义实体
│       ├── repository/
│       │   └── MetadataRepository.java   # 元数据 CRUD
│       └── service/
│           ├── LLMService.java           # LLM 调用（动态 Prompt）
│           ├── SQLExecutorService.java    # SQL 安全执行
│           ├── MetadataService.java       # 元数据管理 + 自动同步
│           └── PromptBuilder.java         # 语义层 Prompt 动态组装
│   └── src/main/resources/
│       ├── application.yml               # 智谱 AI + H2 配置
│       ├── schema.sql                    # 业务表 + 5 张元数据表
│       └── data.sql                      # 销售数据 + 预置语义层
│
└── smart-query-frontend/             # 前端 (Vue 3 + Vite)
    ├── package.json
    ├── vite.config.ts
    └── src/
        ├── App.vue
        ├── main.ts
        ├── views/
        │   └── ChatView.vue              # 对话式查询界面
        └── api/
            └── queryApi.ts               # 后端接口封装
```

## 启动命令

```bash
# 后端 (端口 8080)
cd smart-query-backend && mvn spring-boot:run

# 前端 (端口 5173，需先 npm install)
cd smart-query-frontend && npm run dev
```
